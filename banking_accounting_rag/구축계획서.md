# 은행회계해설 생성형AI 서비스 구축 계획서

## 1. 프로젝트 개요

은행회계해설(상/하) 2018개정판 마크다운 문서를 기반으로, 사용자 질문에 대해 관련 내용을 검색하고 답변하는 **RAG(Retrieval-Augmented Generation) 서비스**를 구축한다.

### 목표
- 은행회계해설 내용에 대한 자연어 질의응답
- 답변 시 출처(상/하권, 페이지 번호) 명시
- **멀티 LLM 백엔드 지원:** 로컬 LLM(Ollama), Gemini API, OpenAI API를 UI에서 선택
- **임베딩은 로컬(bge-m3) 통일**, 벡터 DB 단일 컬렉션으로 공유

### 핵심 구성요소

| 구성요소 | 로컬 모드 (Ollama) | Gemini API 모드 | OpenAI API 모드 |
|---------|-------------------|----------------|----------------|
| LLM (답변 생성) | gemma3:12b | gemini-2.5-flash | gpt-4o |
| 임베딩 (검색) | bge-m3 (공통) | bge-m3 (공통) | bge-m3 (공통) |
| 벡터 DB | ChromaDB 단일 컬렉션 | 동일 | 동일 |
| 비용 | 무료 (로컬) | 무료 티어 (1,500회/일) | 유료 ($2.5~10/1M 토큰) |

**설계 결정:** 임베딩은 로컬 bge-m3로 통일. 검색 품질은 동일하게 유지하면서 LLM만 전환하여 답변 생성 품질을 비교할 수 있다. 벡터 DB를 모드별로 따로 구축할 필요가 없어 시간과 저장공간을 절약한다.

### 3.1~3.5단계: 로컬 RAG 서비스 (완료)

기존 구축 내용과 동일. 구축완료보고서 참조.

### 3.6단계: config.py 확장

**설정 구조:**

```python
# 모드 설정 ("local", "gemini", "openai") — LLM만 전환
MODE = "local"

# 임베딩 (모든 모드 공통)
EMBED_MODEL = "bge-m3"

# LLM 모드별 설정
LOCAL_LLM_MODEL = "gemma3:12b"
GEMINI_LLM_MODEL = "gemini-2.5-flash"
OPENAI_LLM_MODEL = "gpt-4o"

# 벡터 DB (단일 컬렉션)
COLLECTION_NAME = "bank_accounting"
```

- API 키는 `.env` 파일로 관리 (git 미추적)
- UI에서 모드 변경 시 서버 재시작 없이 런타임 전환

### 3.7단계: Gemini API 연동

**추가 패키지:** `google-genai`

| 기능 | 구현 |
|------|------|
| LLM 답변 생성 | `google.genai.Client.models.generate_content()` → 스트리밍 응답 |
| 무료 티어 | 하루 1,500회, RPM 15 |

### 3.8단계: OpenAI API 연동

**추가 패키지:** `openai`

| 기능 | 구현 |
|------|------|
| LLM 답변 생성 | `openai.ChatCompletion.create()` → 스트리밍 응답 |

### 3.9단계: UI 모드 선택 기능

- 헤더에 로컬 / Gemini / OpenAI 라디오 버튼
- 선택 시 `POST /set_mode` API 호출 → 런타임 모드 전환
- API 키 미설정 또는 벡터 DB 미구축 시 해당 모드 비활성화
- 모드 전환 시 채팅 영역에 안내 메시지 표시

**추가 API 엔드포인트:**

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/set_mode` | 모드 변경 (local/gemini/openai) |
| GET | `/get_mode` | 현재 모드 및 사용 가능 모드 조회 |

## 5. 주요 설정값

**`src/config.py`**

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `MODE` | `local` | 기본 LLM 모드 (local/gemini/openai) |
| `OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama 서버 주소 |
| `EMBED_MODEL` | `bge-m3` | 임베딩 모델 (모든 모드 공통) |
| `LOCAL_LLM_MODEL` | `gemma3:12b` | 로컬 LLM 모델 |
| `GEMINI_API_KEY` | 환경변수 (.env) | Gemini API 키 |
| `GEMINI_LLM_MODEL` | `gemini-2.5-flash` | Gemini LLM 모델 |
| `OPENAI_API_KEY` | 환경변수 (.env) | OpenAI API 키 |
| `OPENAI_LLM_MODEL` | `gpt-4o` | OpenAI LLM 모델 |
| `CHUNK_SIZE` | 1000 | 청크 크기 (자) |
| `CHUNK_OVERLAP` | 200 | 청크 오버랩 (자) |
| `TOP_K` | 5 | 검색 결과 수 |
| `COLLECTION_NAME` | `bank_accounting` | ChromaDB 컬렉션명 (단일) |

## 6. 실행 방법

```bash
# 1. 환경 설정
cp .env.example .env
# .env 파일에 API 키 입력 (Gemini/OpenAI 사용 시)

# 2. 패키지 설치
pip install -r requirements.txt

# 3. Ollama 서버 실행 (임베딩 + 로컬 LLM에 필요)
ollama serve

# 4. 벡터 DB 구축 (최초 1회, 약 50분)
cd src
python build_vectordb.py

# 5. 웹 서버 실행
python app.py

# 6. 브라우저 접속 → UI에서 LLM 모드 선택
# http://localhost:5000
```
