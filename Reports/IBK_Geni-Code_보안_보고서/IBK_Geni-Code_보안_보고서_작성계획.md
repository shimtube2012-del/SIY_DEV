  2. **Geni-Code 도입에 따른 내부통제·보안 리스크**를 경영진에게 전달
  3. 관리방안(보안 서약서 등)의 필요성 설득
- **대상 독자**: IBK기업은행 경영진, 보안·내부통제 담당 부서, IT개발 부서장급

- **스타일 가이드(파생)**: `IBK_Geni-Code_보안_보고서_스타일가이드.md` — 원본 PDF 분석 결과(컬러·타이포·컴포넌트·페이지 나눔 규칙) 정리
- **생성 방식**: fpdf2 기반 Python 스크립트 (원본 `C:/SIY_DEV/영일피혁/generate_report.py` 구조 복제)

  - A4 페이지가 넘어갈 때 **문단 중간이나 문장 중간에서 잘리지 않도록** 처리
  - 제목(Heading)은 바로 다음 본문과 **같은 페이지**에 위치 (제목만 페이지 말미에 오는 것 금지)
  - 표·도식은 **단일 페이지 내에 온전히** 배치 (행·셀이 쪼개지지 않도록)
  - 불릿 리스트도 가능한 한 한 페이지 내에서 묶음 유지
  - CSS `page-break-inside: avoid`, `break-inside: avoid`, `orphans/widows` 속성 활용

## 2. 핵심 메시지 (Key Message)

> **"Geni-Code가 생성한 산출물 자체는 기존 통제체계(개발-QA-운영, 결재, 변경관리)로 충분히 보호되나,
> 사용 전제 조건인 업무용 PC의 Python 실행 허용이 통제 사각지대를 만들 수 있다.
> 따라서 기술적 통제(망·시스템)만으로는 부족하며, 사용자 관점의 보안 서약 등 행위적 통제를 병행해야 한다."**

---

## 3. 보고서 목차안 (Outline)

> 분량 제한 없이 내용 완결성을 우선. 각 장은 독립된 논리 단위로 구성하여 페이지 경계에서 잘리지 않도록 작성.

### Ⅰ. 개요
- Geni-Code란: 망분리 환경 하 온프레미스 Qwen Coder 기반 코드 어시스턴트
- 도입 배경 및 기대 효과
- 보고 목적: 시스템 반영 구조 이해 공유 + 리스크 제기 + 대응방안 건의

### Ⅱ. 현행 프로그램 반영 체계 (핵심 장)
- IBK 시스템 구조: 계정계·스마트뱅킹·여신지원 등
- 개발-QA-운영 3단계 프로세스 상세
- 결재 체계: 신규(부장) / 변경(팀장)
- 변경관리 시스템을 통한 소스 관리
- **[도식 1]** 프로그램 작성·반영 흐름도
- 시사점: 작성 주체(사람·Geni-Code·AIDD)와 무관하게 동일 프로세스 적용 → 기존 체계 내 산출물은 통제상 문제 없음

### Ⅲ. 쟁점: 업무용 PC의 통제 사각지대
- 기존 원칙: 업무용 PC는 허가된 프로그램만 실행
- Geni-Code 사용 전제: Python 실행 허용 필요
- 통제 사각지대 발생 메커니즘
- 대표 위협 시나리오 (비인가 자동화, 데이터 유출, 쉐도우IT 등)
- **[도식 2 필요 시]** 기존 통제 vs. Geni-Code 사용 시 차이 비교도

### Ⅳ. 대응 방안 및 건의
- **행위적 통제**: Geni-Code 사용자 보안 서약서 도입 (최우선)
- **기술적 보완**: 실행 환경 제한, 로그 감사, 허용 라이브러리 화이트리스트
- **관리적 보완**: 사용자 교육·주기 점검·위반 시 조치
- 결론 및 즉시 시행 사항

**작성 원칙**: 불릿·표·도식 적극 활용, 경영진 가독성 우선, 각 장이 독립적으로 읽히도록 완결성 확보.

   - Geni-Code 사용 시나리오별 통제 지점 표시도

3. **[3단계] 초안 작성**
   - 위 목차에 따라 본문 작성
   - **`영일피혁_경영보고서_2026.pdf` 레이아웃·서식·톤앤매너 적용** (표지/헤더·푸터/섹션 제목 스타일/표·도식 배치/색상·폰트 등)

4. **[4단계] 검토 및 수정**
   - 내용 검토 → 반영 → 최종본

5. **[5단계] PDF 생성 (fpdf2 기반 Python 스크립트)**
   - `C:/SIY_DEV/영일피혁/generate_report.py`를 복제해 `generate_geni-code_report.py` 신규 작성
   - 유지: `ReportPDF` 클래스, 컬러 상수, 헬퍼(`section_title`, `body_text`, `bullet`, `key_finding`, `add_table`, `callout_box`, 헤더/푸터)
   - 치환: `build_report()` 내부 콘텐츠만 Geni-Code 보고서용으로 교체
   - 제거: 재무 차트 관련 chart_* 함수(이번 보고서에 불필요)
   - 추가: 프로그램 반영 체계 흐름도 PNG(별도 툴로 생성 후 `pdf.image()` 삽입)
   - **페이지 나눔 안전 규칙 (코드 레벨)**:
     - 표 삽입 전: `if pdf.get_y() + 8 + rows*6.5 + 5 > 270: pdf.add_page()`
     - 차트 삽입 전: `if pdf.get_y() + 90 > 270: pdf.add_page()`
     - 콜아웃 삽입 전: `if pdf.get_y() + 25 > 270: pdf.add_page()`
     - 섹션 L1 제목 뒤에는 본문 최소 2줄이 같은 페이지에 오도록 수동 체크
   - 최종 페이지별 육안 검토 (잘림 발생 시 여백·폰트·이미지 크기 조정)
