## 4. 수정 반영 사항

### 수정 4-1: 작성 부서 플레이스홀더 명시
- 현재: 표지의 `작성: [작성 부서명 — 확정 필요]`
- 조치: 초안 상태 유지. PDF 생성 시 **"정보보호부" 또는 사용자 지정 문자열**로 치환 가능하도록 스크립트에 변수화

### 수정 4-2: KF-2 문장 가독성
- 현재 KF-2 마지막 문장이 다소 길다. 경영진 가독성을 위해 한 번 끊는다.
- **수정 전**: "Python 허용은 이 원칙의 예외이자 결재·QA·변경관리를 거치지 않은 프로그램이 업무PC에서 직접 구동될 수 있는 경로를 신설한다."
- **수정 후**: "Python 허용은 이 원칙의 예외다. 결재·QA·변경관리를 거치지 않은 프로그램이 업무PC에서 직접 구동될 수 있는 **새로운 경로가 만들어진다.**"

### 수정 4-3: 위협 시나리오 T3 "공급망 공격" 용어
- "공급망 공격"이라는 용어가 원본 `IBK_바이브코딩_공급망공격_보고서`와 중복되어 보고서 정체성이 흐려질 수 있음.
- **수정**: "공급망 공격" → "외부 패키지 유입 경로의 취약성" 으로 완화 표현 (의미 동일, 중복 회피)

### 수정 4-4: VI장 결론 마지막 문장 강조
- 현재 문장은 좋으나, 결론의 마지막을 **요청 사항으로 명확히** 마무리하는 것이 경영 보고서 관례에 부합.
- **추가 문장**: "이에 **1순위 과제인 보안 서약서 도입을 우선 승인**해 주시기 바랍니다."

### 수정 4-5: 분량·페이지 예상
- 텍스트 6장 + 표 4개 + 도식 2개 → fpdf2 기준 약 **7~9페이지 예상** (표지·목차·Exec.Summary 포함)
- 스타일가이드의 페이지 나눔 규칙으로 잘림 없이 배치 가능

## 5. 수정 적용 완료 후 최종 초안 상태

수정 4-2~4-4는 본 문서 아래에 **최종 반영 텍스트**로 보관하여 5단계 스크립트 이식 시 사용한다.

### [최종 반영] KF-2
> **KF-2. 쟁점은 산출물이 아닌 "실행 환경"에 있음**
> Geni-Code 사용을 위해 업무용 PC에서 **Python 실행을 허용해야 한다.**
> 업무용 PC는 본래 "허가된 프로그램만 실행" 원칙(앱 화이트리스트)으로 통제되어 왔으며,
> Python 허용은 이 원칙의 예외다. 결재·QA·변경관리를 거치지 않은 프로그램이
> 업무PC에서 직접 구동될 수 있는 **새로운 경로가 만들어진다.**

### [최종 반영] T3
> **T3. 외부 패키지 유입 경로의 취약성**
> Geni-Code가 제안한 파이썬 패키지명이 타이포스쿼팅 또는 트로이화된 변종일 경우,
> 내부 미러 경유로 유입된 악성 코드가 업무PC에 상주할 수 있다.
> 실제 PyPI에서 유사 사례가 다수 보고되어 있는 실존 위협이다.

### [최종 반영] VI장 결론 말미 추가 문장
> 이에 **1순위 과제인 보안 서약서 도입을 우선 승인**해 주시기 바랍니다.
