
1. IBK Geni-Code란?
   IBK기업은행은 인터넷망과 업무망이 분리된 망분리 시스템을 운영 중이다
   업무망에서 클로드 코드와 같은 코드 어시스턴트를 쓰기 위해서 내부 업무망에 qwen coder를 이용한 코드 어시스턴트를 온프레미스로 구축했다.

2. IBK 시스템들의 내부통제 및 보안
   계정계, 스마트뱅킹, 여신지원시스템 등 각각의 시스템들은 개발-QA-운영으로 이루어져 있어 개발서버에서 개발이 이루어지고 QA를 통해 검증된 다음 운영에 배포되는 방식으로 되어있다.
   
   이 과정에서 신규 프로그램 생성은 부장 결재, 기존 프로그램은 팀장 결재를 받고 변경관리 시스템에 의해 소스코드가 관리된다.
   
   이러한 구조에서 프로그램 소스를 사람이 만들던, IBK Geni-Code가 만들던, AIDD(외부솔루션)이 만들던 내부통제 및 보안에는 문제가 없다.
   
   (프로그램 작성 및 반영 체계 도식 작업 필요)

2. 무엇이 문제인가?
   IBK Geni-Code를 통해서 작성된 프로그램 소스가 기존 통제체계를 따라서 적용되는 건 내부통제나 보안 등에 아무런 문제가 없다. 다면 현재 직원들의 업무용 PC는 허용된 프로그램만 실행될 수 있고 Geni-Code 사용을 위해서는 업무용 PC에서 파이썬이 실행 가능하도록 보안을 해제해야 한다. 기존 시스템에 기존의 절차를  태우는 것은 문제가 없지만 파이썬이 업무용PC에서 실행 가능하도록 허용해버리게 되면 직원들이 아무런 통제절차나 보안 검증없이 업무용PC에서 작동되는 프로그램을 만들 수가 있게 된다.

3. 어떻게 해야하나?
   AI 시대에 맞게 AI와 일을 해야하는 것은 분명하다. 그렇지만 보안도 중요하다. 기존 시스템은 기존 절차를 따르므로 문제가 없지만 업무용PC에서 직접 작동하는 프로그램을 만드는 건 내부통제와 보안 측면에서 위험한 점이 있다. 이를 해결하기 위해서는 Geni-Code를 사용하는 직원들에게 보안 서약서 등을 받아야 한다.
