Axios npm 공급망 사건으로 바라본 바이브코딩의 유의점

— 비개발자도 이해할 수 있도록 작성 —작성: IBK기업은행 디지털혁신부 · 2026-04-13

1. 보고 배경

최근 "AI에게 말로 시키면 AI가 대신 프로그램을 짜주는" 개발 방식(이른바 바이브코딩, Vibe Coding)이 관심을 받고 있음. 대표 도구가 Claude Code이며, 당행에서는 현재 일부 직원이 개인 노트북에서 실험적으로 써보는 수준임. 행내망과 인터넷망이 물리적으로 분리되어 있어 외부의 위협이 행내 업무시스템까지 직접 전파되기는 어려우나, 2026년 3월 해외에서 발생한 Axios npm 사건은 "AI가 알아서 부품을 가져다 쓰는" 이 방식이 품고 있는 구조적 위험을 잘 보여주었기에, 향후 행내 정식 도입을 검토할 때 미리 알고 있어야 할 사항을 공유하고자 함.

먼저 알아둘 개념 — 프로그램은 모든 기능을 처음부터 짜지 않고, 인터넷의 공용 "부품 창고"에서 조각을 가져와 조립하듯 만듦. 그 창고 중 가장 큰 것이 npm이며, 거기서 가장 많이 쓰이는 부품 하나가 Axios(웹·앱이 서버와 통신할 때 쓰는 기본 부품, 주간 1억+ 회 사용)임. 이번에 누군가 이 인기 부품에 몰래 악성 코드를 심어 창고에 올려버렸고, 이를 받아 쓴 전 세계 개발자 PC가 설치 즉시 원격 조종 프로그램에 감염되어 비밀번호·계정 키가 유출됨.

2. 사건 요약 — Axios npm 사건 (2026.03.31)

누구를 노렸나전 세계 수백만 개발자가 쓰는 가장 인기 있는 "통신 부품" Axios (주간 1억+ 회 사용)
어떻게 침투부품 관리자의 계정을 탈취 → 담당자인 척 악성 버전을 창고에 직접 올림
어떻게 속였나원래 부품 내용은 그대로 두고, 설명서에 이름이 비슷한 가짜 부품(plain-crypto-js) 한 줄만 몰래 끼워넣음 → 기존 보안 검사 도구가 이상을 감지하지 못함
어떻게 작동개발자가 부품을 내려받는 순간 "설치 직후 자동 실행" 기능으로 원격 조종 악성 프로그램이 깔림
피해 범위Windows·Mac·Linux 가리지 않음. 개발자 PC에 저장된 각종 비밀번호·인증서·클라우드 계정 키·환경설정 파일이 외부로 유출되고, 컴퓨터 재부팅 후에도 숨어 있음

3. 바이브코딩이 특별히 더 위험한 이유

핵심: 일반 개발자는 부품을 가져올 때 "이거 써도 될까?" 한 번이라도 살피지만, 바이브코딩은 사람이 "이 기능 만들어줘"라고 말만 하면 AI가 알아서 부품을 고르고 설치까지 진행함. 사람이 마지막으로 확인하는 단계가 사라지는 셈이라, Axios처럼 가짜가 섞여 있어도 사용자가 모른 채 그대로 컴퓨터에 깔릴 수 있음.
  • 사람 점검이 사라짐 — AI가 알아서 설치하므로, 실제로 어떤 부품이 들어왔는지 사용자 본인도 잘 모름
  • 개인 노트북 유출 가능성 — 현재 실험 중인 노트북이 감염될 경우, 그 안에 저장된 개인 이메일·외부 서비스 로그인 정보·업무 메모 등이 외부로 빠져나갈 수 있음
  • 행내 확산 우려(향후) — 정식 도입 후에는 여러 직원의 자동 빌드·배포 시스템까지 같은 경로로 오염될 수 있음
  • 확장 기능이 많음 — Claude Code는 AI에 기능을 덧붙이는 "플러그인"을 손쉽게 설치할 수 있어, 일반 개발 환경보다 위험 진입 경로가 더 많음

4. 행내 도입 시 유의사항 (건의)

[현재 — 개인 실험 단계]

  • 실험 중인 직원에게 업무 계정과 실험용 계정을 분리해 쓰도록 안내
  • 실험용 노트북에 행내 중요 비밀번호·인증서를 보관하지 않도록 권고
  • Axios 같은 실제 사례를 주기적으로 공유해 "AI가 알아서 설치"의 위험성을 지속 인지

[향후 — 행내 정식 도입 시]

  • AI가 새 부품을 설치하거나 명령을 실행할 때는 사람이 먼저 승인하도록 절차 마련
  • 세상에 갓 올라온 부품은 며칠 지나서야 쓸 수 있게 "지연 설치" 규칙 적용 (악성 부품은 대개 몇 시간 내 발견·삭제됨)
  • 공급망 사고를 빠르게 잡아주는 전용 보안 검사 도구를 자동 빌드 과정에 연결
  • 바이브코딩 전용 격리 공간(샌드박스/컨테이너)에서만 사용하도록 표준화
  • AI 플러그인은 사내에서 허용한 목록만 쓰도록 미리 지정

5. 결론

망분리 환경과 실험 단계라는 점에서 지금 당장 큰 사고로 이어질 가능성은 낮음. 그러나 "AI가 알아서 설치"는 곧 "검증되지 않은 코드의 자동 실행"을 의미하며, Axios 사건이 그 위험을 분명히 보여주었음. 향후 바이브코딩을 행내 정식 도입할 때에는 유의사항을 반영한 통제 체계를 먼저 마련한 뒤 단계적으로 확대할 필요가 있으며, 그 전까지는 실험 단계의 보안 인식 제고를 지속해 나가고자 함.
